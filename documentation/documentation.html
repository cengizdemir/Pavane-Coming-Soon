3. Choose a name for your new key and specify the websites on which the key usage will be allowed. If you don’t need any website restriction, just put an * in that field (but don’t leave it blank, unless you are having issues with *! See comments for further information). Then click on Create:




Write down your brand new API key, and click OK




5.Locate the file that contains the Google Maps script code on your website, and append your API key to it as a parameter:

<script src="//maps.googleapis.com/maps/api/js?key=YOUR_API_KEY" async="" defer="defer" type="text/javascript"></script>

6. Your API key should be up and running within 5 minutes.



starlight.js



1.shape

Could be either "circle" or "square"

2.initial_size

The starting size of the stars when they spawn (eg "32px")

3.final_size

The final size of the stars after expansion. Could also be the same as initial size if you don't want the stars to expand (eg "64px")

4.expand_speed

How long the stars take to expand/shrink (in CSS time. eg "1s")

5.fade_delay

The delay until the stars begin to fade out (in CSS time. eg "1s")

6.fade_duration

Possible colors of the stars, each of which will be randomly chosen during spawning. Can be any CSS complient color (eg. "rgba(255,255,255,0.5)")

7.colors

Possible colors of the stars, each of which will be randomly chosen during spawning. Can be any CSS complient color (eg. "rgba(255,255,255,0.5)")

8.frequency

Possible colors of the stars, each of which will be randomly chosen during spawning. Can be any CSS complient color (eg. "rgba(255,255,255,0.5)")

9.density

How many stars will spawn per wave (eg 5)

10.keep_lit

true or false: Whether the stars will fade out

11.rotation

true or false : Whether the stars will rotate

12.coverage

How much of the element's area the stars will show up in (0-1 eg 0.75 will cover 75%)

13.target_class

Elements that the stars will spawn in. Can be any class, element, or id (eg ".starlight")
